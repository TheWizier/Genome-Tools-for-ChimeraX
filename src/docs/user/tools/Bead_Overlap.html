Tool: Overlap Tool

The Overlap Tool makes a new model based on overlaps between existing models.

For this explanation we will be using this example:

We have two models A and B with IDs 1 and 2 respectively. Model A contains bead 1, 2 and 3, model B contains bead 2, 3 and 4. The resulting overlap between model A and B consists of the beads 2 and 3.

The Overlap tool works on a rule based system.

To add a rule click the"+" button.

Each rule has three fields.

The first field is for specifying the overlap. This is specified using comma separated model IDs or model names in double quotes. In our example the overlap between models A and B can be specified as:

1, 2
"A", "B"
or even
"A", 2

The second field is a dropdown menu for specifying what to do with the beads selected in the first field. The option is between colouring them in a certain colour, retaining colours or not including them in the new model. When retaining colours the colours are taken from the first specified model.

The third field is a colour picker for the colour to use for the beads if colouring is selected in the second field.

To remove a rule click that rule's "-" button. To rearrange the priorities of the rules use the arrow buttons.

In the examples "[...]" is used to show a field

Example 1

Colour model A blue, model B red, and the overlap between them green.

Rules in order:

[1, 2][Colour][green]
[1][Colour][blue]
[2][Colour][red]

Example 2

Colour model A blue and model B red but do not include the overlap between them.

Rules in order:

[1, 2][Don't include][]
[1][Colour][blue]
[2][Colour][red]

Example 3

Make a model containing the overlap between A and B and keep the colours from B.

Rules in order:

[2, 1][Retain colours][]
